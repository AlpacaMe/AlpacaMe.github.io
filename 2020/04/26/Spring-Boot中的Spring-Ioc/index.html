Spring Boot

​ Spring Boot是一个基于Spring框架搭建起来的应用，其次它会嵌入Tomcat、Jetty或者Undertow等服务器，并不需要传统的WAR文件进行部署，搭建Spring Boot项目并不需要单独下载Tomcat等传统的服务器；同时提供通过Maven（或者Grandle）依赖的starter，这些starter可以直接获取开发所需要的相关包，通过这些starter项目就能以Java Application的形式运行Spring Boot的项目，而无须其他的配置；对于配置，Spring Boot提供的Spring框架的最大自动化的配置，大量使用自动配置，使得开发者对Spring的配置尽量少些；此外还提供一些监测、自动监测的功能和外部配置，与此同时没有附加代码和XML配置的要求。

2、使用自定义的配置

在src/main/resource/文件夹下面有一个文件application.properties，在该文件夹添加设置就会覆盖原来的原始设置：

server.port=8090
spring.mvc.view.prefix=/WEB-INF/jsp/
spring.mvc.view.suffix=.jsp

然后在控制台就可以看到：Tomcat started on port(s): 8090 (http) with context path ‘’字样表示端口已经从默认的8080端口改到8090端口。

Spring Boot的参数的配置除了使用properties文件之外，还可以使用yml文件等，它还会以下列优先级的顺序进行加载：

  • 命令行参数
  • 来自java:comp/env的JNDI属性；
  • Java系统属性（System.getProperties();）
  • 操作系统环境变量；
  • RandomValuePropertySource配置的 random.*属性值。
  • jar包外部的application-{profile}.properties或application.yml（带spring.profile）配置文件；
  • jar包内部的application-{profile}.properties或application.ym（带spring.profile）配置文件；
  • jar包外部的application.properties或application.yml（不带spring.profile）配置文件；
  • jar包内部的application.properties或application.ym（不带spring.profile）配置文件；
  • @Comfiguration注解类上的@PropertySource；
  • 通过SpringApplication.setDefaultProperties指定的默认的属性。

实际上，yml文件的配置与properties文件只是简写和缩进的区别，因此差异不大。

全注解下的Spring Ioc

Spring最成功的是其提出的理念，而不是技术的本身。它所依赖的两个核心理念：一个是控制反转Ioc（Inversion of Control），另一个就是面向切面编程（Aspect Oriented Programming）。

Ioc容器是Spring的核心，可以说Spring是一种基于Ioc容器的编程框架。因为Spring Boot是基于注解的开发Spring Ioc。Ioc需要具备两个基本的功能：

  • 通过描述管理Bean，包括发布和获取Bean；
  • 通过描述完成Bean之间的依赖关系；

1、Ioc容器简介

Spring Ioc容器是一个管理Bean的容器，在Spring的定义中，它要求所有的Ioc容器都需要实现接口BeanFactory，它是顶级容器接口。

package org.springframework.beans.factory;

import org.springframework.beans.BeansException;
import org.springframework.core.ResolvableType;
import org.springframework.lang.Nullable;

public interface BeanFactory {

           //前缀
    String FACTORY_BEAN_PREFIX = "&";
    //多个getBean()的方法
    Object getBean(String name) throws BeansException;
    <T> T getBean(String name, Class<T> requiredType) throws BeansException;
    Object getBean(String name, Object... args) throws BeansException;
    <T> T getBean(Class<T> requiredType) throws BeansException;
    <T> T getBean(Class<T> requiredType, Object... args) throws BeansException;

         <T> ObjectProvider<T> getBeanProvider(Class<T> requiredType);
    <T> ObjectProvider<T> getBeanProvider(ResolvableType requiredType);
    //是否包含Bean
         boolean containsBean(String name);
        //是否是单例
    boolean isSingleton(String name) throws NoSuchBeanDefinitionException;
        //是否是原型
    boolean isPrototype(String name) throws NoSuchBeanDefinitionException;
        //是否类型匹配
    boolean isTypeMatch(String name, ResolvableType typeToMatch) throws NoSuchBeanDefinitionException;
    boolean isTypeMatch(String name, Class<?> typeToMatch) throws NoSuchBeanDefinitionException;
    @Nullable
         //获取Bean的类型
    Class<?> getType(String name) throws NoSuchBeanDefinitionException;
         //获取bean的别名
    String[] getAliases(String name);

}

  • getBean()方法是Ioc容器中最重要的方法之一，它的意义是从Ioc容器中获取Bean，在Ioc容器中允许按照类型、名称获取Bean。

  • isSingleton()方法判断Bean是否是单例模式的（Spring Ioc容器中，默认的情况下，Bean都是单例存在的，也就是使用getBean方法返回的都是同一个对象）。

  • isProperty()方法与isSingleton()方法相反，如果返回值是true，那么使用getBean方法返回的都是同一个对象时，Spring Ioc容器就会创建一个新的Bean返回给调用者。

    由于BeanFactory的功能还不够强大，因此Spring在BeanFactory的基础上，还有一个更高级的接口AplicationContext。它是BeanFactory的子接口之一，在Spring的体系中BeanFactory和ApplicationContext是最为重要的接口设计。在平常设计中使用大部分的ApplicationContext接口的实现类。

2、Ioc的实例

创建实体类：

package com.panhao.demo.pojo;

import lombok.Getter;
import lombok.Setter;
import lombok.ToString;

@Getter
@Setter
@ToString
public class User {

    private Long id;
    private String userName;
    private String note;

}

然后创建Ioc容器的配置类：

package com.panhao.demo.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import com.panhao.demo.pojo.User;

@Configuration
public class AppConfig {

    @Bean(name = "user")
    public User initUser() {
        User user=new User();
        user.setId(1L);
        user.setUserName("user_name_1");
        user.setNote("note_1");
        return user;
    }
}
  • @Configuration代表这是一个Java配置文件。
  • @Bean代表将initUser方法返回的POJO类装配到SpringIoc容器中，而其属性值name定义这个Bean的名称。如果没有设置该属性，则将方法名称“initUser”作为Bean的名称保存到Spring Ioc中。

创建测试类，利用AnnotationConfigApplicationContext类加载Spring Ioc容器：

package com.panhao.demo.config;
import org.springframework.context.ApplicationContext;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

import com.panhao.demo.pojo.User;

import lombok.extern.slf4j.Slf4j;

@Slf4j
public class IocTest {

    public static void main(String[] args) {
        ApplicationContext ctx=new AnnotationConfigApplicationContext(AppConfig.class);
        User user=ctx.getBean(User.class);
        log.info(user.toString());
    }
}

  • @Slf4j是lombok插件的日志引用，实际上相当于：

    private static Logger log=LoggerFactory.getLogger(IocTest.class);

在控制台观察到输出：11:23:12.232 [main] INFO com.panhao.demo.config.IocTest - User(id=1, userName=user_name_1, note=note_1)

3、Spring Boot装备Bean

Spring允许使用XML配置或者是Java配置文件装配Bean，但是由于Spring Boot是基于注解的方式。所以Spring Boot装配Bean的方式略微不同。

通过扫描装配Bean

如果每一个Bean都是用@Bean配置的话相对来说会很麻烦，使用@Component和@ComponentScan。

  1. @Component ：是标明那个类被扫描进Spring Ioc容器中；
  2. @ComponentScan ：标明采用何种策略去扫描装配Bean；

此时在实体上添加@Component表示这是一个组件，告诉Spring Ico将其装配成Bean：

package com.panhao.demo.config;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;

import lombok.Getter;
import lombok.Setter;
import lombok.ToString;

@Component("user")
@Setter
@Getter
@ToString
public class User {

    @Value("1")
    private Long id;
    @Value("user_name_1")
    private String userName;
    @Value("note_1")
    private String note;

}
  • @Value 表示给这个Bean的各个属性设置一个值。

然后就可以将AppConfig中配置的@Bean删掉，添加@ComponentScan，因为Spring IOC会扫描到User，然后自动装配实例Bean。

package com.panhao.demo.config;

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Configuration;

import com.panhao.demo.pojo.User;

@Configuration
@ComponentScan
public class AppConfig {
}

最后测试这个Bean：

package com.panhao.demo.config;

import org.springframework.context.ApplicationContext;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

import com.panhao.demo.pojo.User;

import lombok.extern.slf4j.Slf4j;

@Slf4j
public class IocTest {

    public static void main(String[] args) {
        ApplicationContext ctx=new AnnotationConfigApplicationContext(AppConfig.class);
        User user=ctx.getBean(User.class);
        log.info(user.toString());
    }
}

在控制台看到如下输出：

10:15:14.743 [main] INFO com.panhao.demo.config.IocTest - UserComponent(id=1, userName=user_name_1, note=note_1)

表示Spring自动装配Bean成功。

@ComponentScan常用配置

  • basePackages :定义扫描的包名；
  • basePackageClasaaes ：定义扫描的类；
  • includeFilters ：定义满足过滤器条件的Bean才会被扫描到；
  • excludeFilter：定义排除满足条件的过滤器之后所有的Bean才会被扫描到；

加入在com.panhao.demo包下有两个类，分别被@Component和@Service所标注，此时不想加载被@Service标注的类加载到IOC容器中，可以这样设置@ComponentScan：

@ComponentScan(basePackages="com.panhao.demo.*",excludeFilter={@Filter(classes={Service.class})})
  • @Service :该注解注入了@Component，所以在默认情况下它会被Spring扫描到Ioc容器中。

为了达到更好的效果可以使用@SpringBootApplication注解，该注解注入了@ComponentScan，此时可以修改上述配置：

@SpringBootApplication
@ComponentScan(basePackages="Com.panhao.demo",exculdeFilter={@Filter(classes=Service.class)})

4、自定义Bean

在Java应用中需要引入很多的第三方的包，并且很有可能希望第三方包的类对象也放到Spring Ioc容器中，此时@Bean注解就发挥很大的作用。

在pom.xml文件中引入相关的依赖：

        <dependency>
            <groupId>org.apache.commons</groupId>
            <artifactId>commons-dbcp2</artifactId>
        </dependency>
        <dependency>
            <groupId>mysql</groupId>
            <artifactId>mysql-connector-java</artifactId>
        </dependency>

然后就可以直接在AppConfig.class文件中定义一个Bean：

package com.panhao.demo.config;


import java.util.Properties;

import javax.sql.DataSource;

import org.apache.commons.dbcp2.BasicDataSource;
import org.apache.commons.dbcp2.BasicDataSourceFactory;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Configuration;



@Configuration
@SpringBootApplication
@ComponentScan(basePackages = "com.panhao.demo")
public class AppConfig {

    @Bean(name = "dateSource")
    public DataSource getDataSource() {
        Properties props=new Properties();
        props.setProperty("driver", "com.mysql.jdbc.cj.Driver");
        props.setProperty("url", "jdbc:mysql://localhost:3306/mybatis?useSSL=false&serverTimezone=UTC");
        props.setProperty("username", "root");
        props.setProperty("password", "panhao");
        DataSource dataSource=null;
        try {
            dataSource=BasicDataSourceFactory.createDataSource(props);
        } catch (Exception e) {
            // TODO Auto-generated catch block
            e.printStackTrace();
        }
        return dataSource;
    }
}

可以 测试文件中通过获取这个Bean测试Spring Ioc是否实例化了这个Bean。

5、依赖注入

除了上述的装备Bean，Spring Ioc还有一个主要的作用就是依赖注入（DI）。

假设人类依赖于动物做事情，先定义一个人类接口和一个动物接口：

package com.panhao.demo.pojo;

public interface Person {

    //使用动物服务
    public void service();

    public void setAnimal(Animal animal);
}

package com.panhao.demo.pojo;

public interface Animal {
    public void use();
}

然后设置人和动物的实现类，实现注入：

package com.panhao.demo.pojo;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;

@Component
public class BussinessPerson implements Person{

    @Autowired
    private Animal animal;

    public void service() {
        // TODO Auto-generated method stub
        this.animal.use();
    }

    public void setAnimal(Animal animal) {
        // TODO Auto-generated method stub
        this.animal=animal;
    }

}

package com.panhao.demo.pojo;

import org.springframework.stereotype.Component;

@Component
public class Dog implements Animal{

    public void use() {
        // TODO Auto-generated method stub
        System.out.println("狗"+Dog.class.getSimpleName()+"看门用的。");
    }

}
  • @Autowired 是Spring中常用的注解之一，十分重要。它会根据属性的类型找到对应的Bean进行注入。这里Dog是动物的一种，所以Spring Ioc容器就会把Dog的实例注入到BussinessPerson中。

测试注入的Bean是否成功：

package com.panhao.demo.config;

public class IocTest {

    public static void main(String[] args) {
        ApplicationContext ctx=new AnnotationConfigApplicationContext(AppConfig.class);
        Person person=ctx.getBean(BussinessPerson.class);
        person.service();
    }
}

在控制可以看到输出：狗Dog看门用的。

@Autowired注解

当只有一个Animal实现类的时候，这么写Spring Ioc容器会正常运行。当拥有两个Animal的实现类的时候，Spring Ioc就会因为没有指定注入哪一个Bean而发生异常：

No qualifying bean of type 'com.panhao.demo.pojo.Animal' available: expected single matching bean but found 2: cat,dog

可以通过修改BussinessPerson中属性的名称来表示要注入的是cat还是dog

@Autowired
private Animal dog;//表示注入的是dog

@Autowired首先它会根据类型找到对应的Bean，如果对应类型的Bean不是唯一的，那么它会根据Bean的名称进行匹配。如果匹配上得上，就会使用该Bean；如果无法匹配就会报出异常。

其次@Autowired是一个必须找到匹配类型Bean的注解，如果没有找到就会抛出异常。如果不是非常确定标注的属性是否是存在的可以使用它的required属性，来允许它为空：@Autowired(required=false)

@Primary和@Qualifier

  • @Primary 是一个修改优先权的注解，当有猫有狗的时候，假设需要猫，就在猫类的定义上添加@Primary就可以了。

但是当@Primary在多个类上做了注解的时候，Spring Ioc依旧是不知道要是用那个类注入。此时，使用@Qualifier提供了更大的灵活性。

  • @Qualifier 它的配置项value需要一个字符串去定义，它将于@Autowired组合在一起，通过类型和名称一起找到Bean。

    @Autowired
@Qualifier(value="dog")
private Animal animal=null;

带有参数的构造方法类的装配

使用@Autowired对构造方法的参数进行注入。

package com.panhao.demo.pojo;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.beans.factory.annotation.Qualifier;
import org.springframework.stereotype.Component;

@Component
public class BussinessPerson implements Person{

    private Animal animal=null;

    public BussinessPerson( @Autowired @Qualifier("dog") Animal animal) {
        this.animal=animal;
    }

    public void service() {
        // TODO Auto-generated method stub
        this.animal.use();
    }

    public void setAnimal(Animal animal) {
        // TODO Auto-generated method stub
        this.animal=animal;
    }

}

对构造方法参数进行注入，在每次构造一个对象的时候都会自动的注入Animal，@Qualifier注解的使用时为了避免歧义性。

6、Bean的生命周期

一个Bean的生命周期包括：Bean的定义，Bean的初始化，Bean的生存期和Bean的销毁。

在@ComponentScan中有一个lazeinit的配置项，用于延迟加载Bean，只有在用到的时候才去加载这个Bean。其值的类型为Boolean，默认值为False，表示在扫描的到时候就进行加载。设置为True，表示延迟加载。

@ComponentScan(basePackages="com.panhao.demo.*",lazyinit=true)

对于第三方的Bean，@Bean注解提供了关于生命周期函数的设置initMethod以及destoryMethod表示在生命周期函数中的哪一个函数进行初始化，哪一个函数进行销毁。

@Bean(initMethod="init",destoryMethod="destory")

  使用@Value注解和`${属性名}`的形式调用属性值。

8、条件装配

有些时候某些客观因素会使一些Bean无法进行初始化。例如，配置数据源的时候，配置中漏掉一些配置会造成数据源不能连接上。在这样的情况下，Ioc容器如果还进行数据源的装配，就会导致应用无法继续。此时希望Ioc容器不去装配数据源。

使用@Conditional注解，它需要配置另外一个接口完成Condition（org.springframework.context.annotation.Condition）来完成对应的功能。

首先创建DatabaseConditional.class，当环境配置不齐全的时候，不实例化这个Bean：

package com.panhao.demo.config;

import org.springframework.context.annotation.Condition;
import org.springframework.context.annotation.ConditionContext;
import org.springframework.core.env.Environment;
import org.springframework.core.type.AnnotatedTypeMetadata;

public class DatabaseConditional implements Condition{

    public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
        // TODO Auto-generated method stub
        //取出环境配置
        Environment env=context.getEnvironment();

        return env.containsProperty("jdbc.driverName")
                &&env.containsProperty("jdbc.url")
                &&env.containsProperty("jdbc.username")
                &&env.containsProperty("jdbc.password");
    }

}

实现Condition接口，需要重写matches()方法，当match方法返回值为true的时候才会装配数据库连接池。

然后修改Bean的注入，加上@Conditional注解：

package com.panhao.demo.config;


import java.util.Properties;

import javax.sql.DataSource;

import org.apache.commons.dbcp2.BasicDataSource;
import org.apache.commons.dbcp2.BasicDataSourceFactory;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Conditional;
import org.springframework.context.annotation.Configuration;



@Configuration
@SpringBootApplication
@ComponentScan(basePackages = "com.panhao.demo")
public class AppConfig {

    @Bean(name = "dateSource",destroyMethod = "close")
    @Conditional(value =DatabaseConditional.class )
    public DataSource getDataSource(
            @Value("{$jdbc.driverName}")String driver,
            @Value("${jdbc.url}")String url,
            @Value("${jdbc.username}")String username,
            @Value("${jdbc.password}")String password) {
        Properties props=new Properties();
        props.setProperty("driver", driver);
        props.setProperty("url", url);
        props.setProperty("username", username);
        props.setProperty("password", password);
        DataSource dataSource=null;
        try {
            dataSource=BasicDataSourceFactory.createDataSource(props);
            System.out.println("正在装配DataSource");
        } catch (Exception e) {
            // TODO Auto-generated catch block
            e.printStackTrace();
        }
        return dataSource;
    }
}

在控制台可以看到输出：正在装配DataSource；当将jdbc.properties中任意一个配置注释掉的时候，控制台将不会输出这句话，表示条件不符合没有装配实例Bean。

9、Bean的作用域

在BeanFactory的源码中存在：isSingleton()和isProperty()两个方法，用来判断Bean的类型。在Spring 应用中Bean有两种类型，一种是单例，即实例化的Bean在Spring Ioc容器中都是唯一的存在；一种是原型，即实例化的Bean，Spring Ioc容器新建的。

设置Bean的模式的在Spring中使用的是：@Scope()注解。

package com.panhao.demo.pojo;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.beans.factory.config.ConfigurableBeanFactory;
import org.springframework.context.annotation.Scope;
import org.springframework.stereotype.Component;

@Component
//@Scope(ConfigurableBeanFactory.SCOPE_PROTOTYPE)
public class ScopeBean {

    @Value("name_Scope")
    private String name;
}

首先注释掉@Scope采用默认的单例模式：

package com.panhao.demo.config;

import javax.sql.DataSource;

import org.springframework.context.ApplicationContext;
import org.springframework.context.annotation.AnnotationConfigApplicationContext;

import com.panhao.demo.pojo.BussinessPerson;
import com.panhao.demo.pojo.Person;
import com.panhao.demo.pojo.ScopeBean;
import com.panhao.demo.pojo.User;

import lombok.extern.slf4j.Slf4j;

@Slf4j
public class IocTest {

    public static void main(String[] args) {
        ApplicationContext ctx=new AnnotationConfigApplicationContext(AppConfig.class);
        ScopeBean sb1=ctx.getBean(ScopeBean.class);
        ScopeBean sb2=ctx.getBean(ScopeBean.class);
        System.out.println(sb1==sb2);
    }
}

控制台输出字样：true。如果不注释@Scope，会发现控制台输出为false。表示设置Scope为Prototype的时候，每次调用这个Bean的时候Spring Ioc容器都会新建一个实例以供使用。

10、@Profile的使用

@Profile的使用，是用来指定不同环境下的配置文件的。比如现在由三个各不同的环境下的配置文件：dev、env、test。然后对应的配置文件为：application-dev.properties、application-test.properties、application-env.properties;

可以在application.properties中指定对应的当前环境：

spring.profiles.active=test

然后需要指定的代码的环境下使用@Profile:

package com.panhao.demo.config;


import java.util.Properties;

import javax.sql.DataSource;

import org.apache.commons.dbcp2.BasicDataSource;
import org.apache.commons.dbcp2.BasicDataSourceFactory;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.Conditional;
import org.springframework.context.annotation.Configuration;



@Configuration
@SpringBootApplication
@ComponentScan(basePackages = "com.panhao.demo")
public class AppConfig {

    @Bean(name = "dateSource",destroyMethod = "close")
    @Profile("dev")
    public DataSource getDataSource(
            @Value("{$jdbc.driverName}")String driver,
            @Value("${jdbc.url}")String url,
            @Value("${jdbc.username}")String username,
            @Value("${jdbc.password}")String password) {
        Properties props=new Properties();
        props.setProperty("driver", driver);
        props.setProperty("url", url);
        props.setProperty("username", username);
        props.setProperty("password", password);
        DataSource dataSource=null;
        try {
            dataSource=BasicDataSourceFactory.createDataSource(props);
            System.out.println("正在装配DataSource");
        } catch (Exception e) {
            // TODO Auto-generated catch block
            e.printStackTrace();
        }
        return dataSource;
    }
}

Spring Ioc会去application.properties中查询被激活的是哪一个环境，然后就会去读取相应的配置文件实例化这个Bean。

11、使用Spring El表达式

通过Spring EL可以拥有更强大的运算规则来更好的装配Bean。

最常用的就是使用${...}的形式读取属性文件的值

@Value("${jdbc.username}")
private String username;

除此之外它还可以调用方法

@Value("#{T(System).currentTimeMillis()}")
private Long  initTime=null;
  • #{...} 代表启用Spring的表达式，它将具有运算的功能；T{...} 代表引入的是类；（System是Java.lang.*包的类，这是Java默认加载的包，因此可以不必写全限定名，如果是其他的包，则需要写出全限定名才能引用类；）currentMillis是System的静态的方法，也就是调用一次System.currentMillis()方法来返回当前时间的毫秒值为这个属性赋值。

此外还可以还可以直接给属性赋值

//赋值字符串
@Value("#{'使用Spring EL 赋值字符串'}")
private String str=null;
//科学计数法赋值
@Value("#{9.3E3}")
private double d;
//赋值浮点数
@Value("#{3.14}")
private float pi;

甚至有时候可以使用其他的Bean的属性来给当前Bean赋值：

@Value("#{beanName.str}")
private String otherBeanProp= null;
  • 这里的BeanName是在Spring Ioc容器中注册的Bean的name属性的值。

还可以使用?来表示判断这个属性是否为空：

@Value("#{beanName.str?.toUpperCase()}")

当这个属性不为空的时候，就可以将其全部变成大写的。

除了以上所有的功能外，最主要的是计算功能

//数学运算
@Value("#{1+2}")
private int run;
//浮点数比较运算
@Value("#{beanName.pi==3.14f}")
private boolean piFlag;
//字符串比较运算
@Value("#{beanName.str eq 'Spring Boot'}")
private boolean srtFlag;
//字符串连接
@Value("#{beanName.str + 'String 连接字符串'}")
private String strApp=null;
//三元运算
@Value("#{beanName.d>100?'大于':'小于'}")
private String resultDesc=null;

常用的Spring EL表达式就是上述这些。
