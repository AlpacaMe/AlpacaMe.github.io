组件

1、组件名

在注册一个组件的时候必须为它取一个组件名称：

Vue.component("my-component-name",options);

组件名的命名方式有两种:使用kebad-case、使用PascalCase

  • Kebad-case :当使用 kebab-case (短横线分隔命名) 定义一个组件时，你也必须在引用这个自定义元素时使用 kebab-case，<my-component-name>

    Vue.component('my-component-name', { /* ... */ });

  • PascalCase :当使用 PascalCase (首字母大写命名) 定义一个组件时，你在引用这个自定义元素时两种命名法都可以使用。也就是说 <MyComponentName> 和 <my-component-name> 都是可接受的。尽管如此，直接在 DOM (即非字符串的模板) 中使用时只有 kebab-case 是有效的。

2、注册组件及其使用

为了能够在模板中使用组件，这些组件必须先注册以便Vue能够识别。这里有两种组件注册的类型：全局注册、局部注册

全局注册

使用Vue.component(conponent-name,options)构造器注册一个组件：

  1. component-name:表示组件的名称；
  2. options ：因为组件是可以复用的Vue实例，所以可以接受与Vue相同的选项，例如：data、computed、watch、methods。

全局注册的组件可以用在其被注册之后的任何 (通过 new Vue) 新创建的 Vue 根实例，也包括其组件树中的所有子组件的模板中

组件的使用，直接<component-name>标签就可以了

<component-name></component-name>
局部注册

全局注册的方式虽然是很可行的，但是在一些组件并不需要在所有的Vue的实例中去运行，所以局部注册的方式在资源的节约上面有很独到的作用。局部注册就是通过一个普通的JavaScript对象来定义一个组件。

var ComponentA ={
    data:function(){
        return {
            count:0
        }
    },
    template:`<button v-on:click="count++">You clicked me {{count}} times.</button>`
}

var Vue({
    el:"#app",
    conponents:{
        'component-A':componentA
    }
})

对于 components 对象中的每个属性来说，其属性名就是自定义元素的名字，其属性值就是这个组件的选项对象。

局部注册的组件在其子组件中不可用，如果非要在子组件中使用，需要在子组件中也注册一次：

var ComponentA={/****/}
var ComponentB={
    components:{
        'component-a':ComponentA
    }
}

var Vue({
    el:"#app",
    components:{
        'component-a':ComponentA,
        'component-b':ComponentB,
    }
})
基本示例
<div id="components-demo">
  <button-counter></button-counter>
</div>
// 定义一个名为 button-counter 的新组件
Vue.component('button-counter', {
  data: function () {
    return {
      count: 0
    }
  },
  template: '<button v-on:click="count++">You clicked me {{ count }} times.</button>'
})
//创建一个Vue的实例
new Vue({ el: '#components-demo' })

与Vue实例所不同的是，Component的data必须是一个函数，因此每一个组件可以维护一份被返回对象的独立拷贝。

组件的复用

组件可以像其他的HTML元素一样进行任意次数的复用：

<div id="components-demo">
  <button-counter></button-counter>
  <button-counter></button-counter>
  <button-counter></button-counter>
</div>

3、通过prop向子组件传递数据

Prop是在组件注册的时候自定义的一些属性。 当一个值传递给Prop的时候，它就变成了组件实例的一个属性。

Vue.component('blog-post',{
    props:['title'],
    templete:'<h3>{{titile}}</h3>'
})
<blog-title title="My Prop name is title" class="text-primary mt-3 "></blog-title>

一个组件默认可以拥有任意属性的Prop属性，任何值都可以传递给Prop。

也可以使用动态的传递数据的方式给Prop属性赋值。

<blog-post
  v-for="post in posts"
  v-bind:key="post.id"
  v-bind:title="post.title"
></blog-post>
new Vue({
  el: '#blog-post-demo',
  data: {
    posts: [
      { id: 1, title: 'My journey with Vue' },
      { id: 2, title: 'Blogging with Vue' },
      { id: 3, title: 'Why Vue is so fun' }
    ]
  }
})
Props的数据类型

props的数据类型不仅仅是字符串数组的形式，通常每一个props的数据类型都可自定义，通过对象的形式：

Vue.component('component-demo',{
    props:{
        title:String,
        likes:Number,
        isPublished:Boolean,
        commentIds:Array,
        author:Obejct,
        callback:Function,
        contactsParams:Promise
    }
})
单向数据流

所有的Prop都使得父子prop之间形成一个单向下行绑定：父级prop更新会向下流动到子组件中，但是反过来则不行。这样会防止从子组件意外改变父级组件的状态，从而导致应用的数据流向难以判断。额外的，每次父级props更新的时候，子组件中的所有的prop都将会刷新为最新的值。

常见的改变props的方式

  1. 这个prop用来传递初始值，子组件希望将其作为本地的prop数据使用。在这种情况下，最好定义一个本地的data属性并将这个prop用作初始值。

    props:["initiaCounter"],
data:function(){
    return {
        counter:this.initiaCounter
    }
}

  2. 这个prop以一种原始的值传入且需要进行转换。在这种情况下最好使用这个prop值来定义一个计算属性：

    props:["size"],
computed:{
    normalizedSize:function(){
        return this.size.trim().toLowerCase()
    }
}
prop数据验证

可以为组件的prop指定验证的要求，可以为props中的值提供一个带有验证需求的对象，而不是一个字符串数组。

Vue.component('my-component', {
  props: {
    // 基础的类型检查 (`null` 和 `undefined` 会通过任何类型验证)
    propA: Number,
    // 多个可能的类型
    propB: [String, Number],
    // 必填的字符串
    propC: {
      type: String,
      required: true
    },
    // 带有默认值的数字
    propD: {
      type: Number,
      default: 100
    },
    // 带有默认值的对象
    propE: {
      type: Object,
      // 对象或数组默认值必须从一个工厂函数获取
      default: function () {
        return { message: 'hello' }
      }
    },
    // 自定义验证函数
    propF: {
      validator: function (value) {
        // 这个值必须匹配下列字符串中的一个
        return ['success', 'warning', 'danger'].indexOf(value) !== -1
      }
    }
  }
})
非Props的属性

组件可以接受任意的属性，而这些属性会被添加到这个组件的根元素上：

<bootstrap-date-input data-date-picker="actived"></bootstrap-date-input>

此时data-date-piceker="active"就会自动添加到<bootstrap-date-input>的根元素上。

属性之间的替换和合并

假设，<bootstrap-date-input>的模板如下：

<input  type="date" class="form-control" >

在运行该组件的时候为其添加合适的class：

<bootstrap-date-input data-date-picker="active" class="date-picker-theme-dark" ></bootstrap-date-input>

对于大多数的属性来说，从外部提供给组件的值会替换掉内部设置好的值，所以此时传入的值为type="text"的时候，模板中的type="date"就会被破坏掉。但是，与之不同的是class和style属性，这两个属性是相互合并，上述使用中，最终的class属性为form-control date-picker-theme-dark。

禁用Attribute的继承

在组件的选项中设置inheritAttrs:false即可以禁用属性继承到组件的根元素上。

Vue.component('my-component',{
    inheritAttrs:false,
    props:['label','value'],
    template:
    `<label>
    {{label}}
    <input v-bind="$attrs" 
            v-bind:value="value"
            v-on:input="$emit('input',$event.target.value)"
    ></label>`
})

4、单个根元素

单个根元素的意思是在声明的时候，template属性中必须只有一个根元素。例如：template:"<h3>Vue组件基础</h3><div v-html="content"></div> 此时Vue就会报错。此时的解决的办法就是，使用一个根元素包裹他们：

<div class="blog-post">
    <h3> {{title}}</h3>
    <div v-html="content"></div>
</div>

当Props的属性增多的时候，为每一个元素设置一个props属性是非常麻烦的一件事。

<blog-post v-for="post in posts" v-bind:key="post.id" v-bind:post="post"></blog-post>
Vue.component('blog-post',{
    props:['post'],
    template:`
    <div class="blog-post">
        <h3>{{title}}</h3>
        <div v-html="post.content"></div>
    </div>
    `
})

此时修改props属性的中的值为：["post"] 就可以变成对象，然后使用.引用。

5、监听子组件事件

new Vue({
  el: '#blog-posts-events-demo',
  data: {
    posts: [/* ... */],
    postFontSize: 1
  }
})
    Vue.component('blog-post', {
        props: ['post'],
        template: `
    <div class="blog-post">
      <h3>{{ post.title }}</h3>
      <div v-html="post.content"></div>
      <button v-on:click="$emit('enlarge-text')">Enlarge text</button>
    </div>
  `
    })

在组件中设置v-on:enlarge-text自定义一个事件来处理这事情，v-on可以监听子组件实例的任意事件。

        <div :style="{ fontSize: postFontSize + 'em' }">
            <blog-post  
                       v-on:enlarge-text="postFontSize += 0.1" 
                       v-for="post in posts" 
                       v-bind:key="post.id" 
                       v-bind:post="post" 
                       class="text-info"></blog-post>
        </div>

同时子组件可以通过调用内建的 $emit(event-name)并传入事件名称来触发一个事件：

<button v-on:click="$emit('enlarge-text')">
  Enlarge text
</button>
使用事件抛出一个值

有时候使用一个特定的时间抛出一个值是很常见的做法。此时可以使用$emit(event-name,params)第二个参数来决定。

<button v-on:click="$emit('enlarge-text', 0.1)">
  Enlarge text
</button>

然后在父组件中使用$event访问到被抛出的值：

<blog-post
  ...
  v-on:enlarge-text="postFontSize += $event"
></blog-post>

如果处理这个事件是一个函数，那么抛出来的值就为作为函数的第一个参数传入这个方法中：

<blog-post
           ....
           v-on:elarge-text="onEnlargeText"
           ></blog-post>
methods:{
    onEnlargeText:function(enlargeAmount){
        this.postFontSize+=enlargeAmount;
    }
}
在组件上使用v-model

自定义事件也可以创建支持v-model的自定义输入组件。

<custom-input  v-model="searchText"></custom-input>

为了让v-model可以正常的工作。必须使得：

  • 将其value属性绑定到一个名叫value的prop上;
  • 在其input事件被触发的时候，将新的值通过自定的input事件抛出去
Vue.component('custom-input',{
    props:['value'],
    template:`
    <input v-bind:value="value" v-on:input="$emit('input',$event.target.value)"
`
})

此时serchText所绑定的数值就是从子组件中冒泡出来的。

将原生事件绑定到组件

在组件上监听原生事件需要用到v-model的native属性。

<base-input v-on:focus.native="onFocus"></base-input>

但是这种在监听一个根节点为<lable>的模板组件时候并不管用。Vue提供$listeners属性，它是一个对象，该对象中包含了作用在这组件上所有的监听器。

{
    focus:function(event){/****/},
    input:function(event){/****/}
}

此时可以通过$listeners属性进行分发监听器：

Vue.component('base-input',{
    inheritAttrs:false,
    props:["label","value"],
    computed:{
        inputListeners:function(){
            var vm=this;
             // `Object.assign` 将所有的对象合并为一个新对象
            return Object.assign({},
                                      // 我们从父级添加所有的监听器
                                 this.$listeners,
                                         // 然后我们添加自定义监听器，
                        // 或覆写一些监听器的行为
                                 {
                            input:function(event){
                        vm.$emit('input',event.target.value)
                           }
            })
        }
    },
    template:
    `<label>
    {{label}}
    <input 
        v-bind="$attrs"
        v-bind:value="value"
        v-on="inputListeners">
    </label>`
})

6、通过插槽分发内容

当向一个组件内传递内容的时候可以使用<slot>标签，使用方式形如：<p>hello</p>，此时组件的模板为：

Vue.component('alert-box' ,{
    template:`<div class="demo-alert-box">
    <strong>Erro!</strong>
    <slot></slot>
</div>`
})
<alert-box>Something bad hanppend. </alert-box>
编译的作用域

当在插槽中运用数据的时候，插槽模板跟其他地方一样可以访问相同的实例属性，而不能访问父级元素的属性。父级模板里所有的内容都是在父级作用域中编译的；子模板里所有的内容都是在子作用域中编译的。

插槽的后备内容

后备内容会在插槽没有提供内容的时候被渲染出来。

Vue.component('submit-button',{
    template:
    `<button type="submit"><slot>Submit<slot></button>`
})

此时在父级元素的未填充的情况下,<submit-button></submit-button>依旧会显示Sumbit按钮。

但是当填充内容的时候，submit就会被填充的内容取代，<submit-button>Save</submit-button>
