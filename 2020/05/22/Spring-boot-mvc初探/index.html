Spring Mvc框架的设计

1、 Mvc框架流程

Mvc框架的执行流程：

  1. 处理请求首先达到控制器。控制器的作用是进行请求分发，这样它会根据请求的内容去访问模型层。
  2. 现今互联网数据主要从NoSQL中来，而且对于数据库而言往往还是数据库的事务机制，为了适应这样的变化，设计者往往会把模型层细分为两层，即服务层Service和数据访问层DAO。
  3. 当控制器获取到由模型层返回的数据的后，就将其渲染到视图中，这样就可以展示给用户了。

2、Spring Mvc流程

![](F:\blog\AlpacaMe\source_posts\Spring Mvc.png)

严格的来说，Spring Mvc运行并非是需要经过所有的流程，@ResponseBody注解的时候就不会进行ViewResolver这个流程。在Web服务启动的过程中，如果在Spring Boot机制下启用Mvc，它就开始初始化一些重要的组件，如DisPatcherServlet、HandlerAdapter的实现类RequestmappingHanlder 等。

3、Spring Mvc开发流程

创建一个UserController.java作为测试的控制器：

package com.panhao.controller;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.ResponseBody;
import org.springframework.web.servlet.ModelAndView;

import com.panhao.pojo.User;

@Controller
@RequestMapping("/user")
public class UserController {


    @RequestMapping("/get")
    @ResponseBody
    public User getUser() {
        User user=new User();
        user.setUsername("username");
        user.setAge(22);
        return user;
    }
}

然后启动Spring Boot项目，在浏览器中输入http://localhost:8080/user/get

就可以在页面看到返回值为：{"username":"username","age":22,"note":null}

  1. @Controller 表明这是一个控制器，然后@RequestMapping 代表请求的路径和控制器（或其方法）的映射关系，它会在web服务器启动Spring Mvc时，就被扫描到HandlerMapping的机制中存储，之后在用户发起请求的时候被DispatcherServlet拦截后，通过URL和其它的条件，通过HandlerMapper机制就能找到对应的控制器（或其方法）进行响应。只是通过HandlerMapping返回的是一个HandlerExecutionChain对象。

    //拦截器列表

}

    HandlerExecuteChain对象包含一个Handler，这里的处理器是对Controller的包装。因为控制器方法可能存在参数，那么处理器就可以读入HTTP和上下文的相关参数，然后在传递给控制器方法。而控制器返回后，处理器又可以通过配置信息对控制器的返回结果进行处理。

  2. 在处理器调用控制器时，它首先用过模型层得到数据，再放入模型层中，最后将返回模型和视图（ModelAndView）对象，这里控制器设置的视图名称是user/details。

配置视图解析器

虽然说Spring Boot在启动Web服务器的时候会执行ViewResolver的自动初始化执行。当然也可以定制InternalResourceViewResolver初始化，可以在配置文件中application.properties中进行配置。

spring.mvc.view.prefix=/WEB-INF/jsp/
spring.mvc.view.suffix=.jsp

进行上述的修改之后，在控制器中返回是user/details，那么它就会找到/WEN-INF/jsp/user/details.jsp 作为视图。

视图解析器定位到视图后，视图的作用是将数据模型渲染，这样就能够响应用户的请求。这一步就是视图将数据模型渲染出来，用来展示给用户查看。

通过MappingJackson2JsonView转换出JSON

目前前后台分离的趋势，使用JSON已经是主流的方式，正如之前使用@ResponseBody标注的方法一样，在后面Spring Mvc会把数据转换为JSON数据集，但是在这里暂时不谈论@ResponseBody ，因为它会采用处理器内部的机制。使用MappingJackson2JsonView 也可以达到最终的效果。

@Controller
@RequestMapping("/user")
public class UserController {

    @RequestMapping("/detailForJson")
    public ModelAndView detailsForJson(Integer id) {
        User user=new User();
        user.setUsername("username
        user.setAge(22);
        user.setNote("note")
        ModelAndView mv=new ModelAndView();
        MappingJackson2JsonView jsonView=new MappingJackson2JsonView();
        mv.setView(jsonView);
        mv.addObject("user",user);
        return mv;
    }
}

在浏览器输入http://localhost:8080/user/detailForJson 将返回一个JSON的字符串：

{"user":{"username":"username","age":22,"note":"note"}}

  • 在控制器方法中模型和视图（ModelAndView）中捆绑了JSON视图（MappingJackson2JsonView）和数据模型（User对象），然后返回，其结果也会转变为JSON，只是需要注意的是这一步与使用JSP作为视图是不一样的，是因为JSON视图是没有视图解析器的定位视图，它不是一个逻辑视图，只是需要将数据模型转为JSON对象而已。

    @ResponseBody则是在一个处理器内部机制转换的

4、Spring Mvc深入开发

处理器映射

如果Web工程使用了Spring Mvc，那么它在启动阶段就会将注解@RequestMapping所配置的内容保存到处理器映射（HandlerMapping）机制中去，然后等待请求的到来，通过拦截请求信息与HandlerMapping进行匹配，找到对应的处理器（它包含控制器的逻辑），并将处理器及其拦截器保存到HandlerExecuteChain对象中，返回给DispatcherServlet，这样DispatcherServlet就可以运行它们了。

关于@RequestMapping的配置如下的源码所示：

@Target({ElementType.TYPE, ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Mapping
public @interface RequestMapping {
    //配置请求映射的名称
    String name() default "";
    //通过路径映射
    @AliasFor("path")
    String[] value() default {};
    //通过路径映射回path配置项
    @AliasFor("value")
    String[] path() default {};
    //限制只响应HTTP请求类型，如GET、POST、HEAD、OPTIONS、PUT、TRACE等
    RequestMethod[] method() default {};
    //当存在对应的HTTP参数时才响应请求
    String[] params() default {};
    //限制请求头存在对应的参数的时候才响应
    String[] headers() default {};
     //限定HTTP请求体提交的类型，如application/json、text/html
    String[] consumes() default {};
    //限定返回的内容类型，仅当HTTP请求头中的Accept类型中包含该指定的类型时才返回
    String[] produces() default {};

}
  1. 可以通过配置value和path来设置请求的URL。配置项value和path可以通过正则表达式来让方法匹配多个请求（从现实的角度上将，如果不是有必要，尽量不要这么做）。
  2. method配置项可以限定HTTP请求的类型，可以区分HTTP的GET或者是POST等不同的请求。（在Spring4.3版本以后，为了简化method配置项的配置，新增了几个注解，如@GetMapping、@PostMapping、@PatchMapping、@PutMapping和@DeleteMapping）。
控制器获取参数

处理器是对控制器的包装，在处理器运行的过程中会调度控制器的方法，只是它在进入控制器方法之前会对HTTP的参数和上下文进行解析，将它们转换为控制器所需要的参数。

在无注解下获取参数

在没有注解的情况下，Spring Mvc也可以获取参数，且参数允许为空，唯一的要求是参数的名称和HTTP请求的参数名保持一致。

package com.panhao.controller;

import java.util.HashMap;
import java.util.Map;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.ResponseBody;

@Controller
@RequestMapping(value = "/my")
public class MyController {

    @GetMapping("/no/annotation")
    @ResponseBody
    public Map<String, Object> noAnnotation(Integer intVal,Long longVal,String str){
        Map<String, Object> paramsMap =new HashMap<String, Object>();
        paramsMap.put("intVal", intVal);
        paramsMap.put("longVal", longVal);
        paramsMap.put("str", str);
        return paramsMap;
    }
}

启动SpringBoot项目之后再浏览器输入地址：http://localhost:8080/my/no/annotation?intVal=10&longVal=200

然后在浏览看到如下返回信息：{"str":null,"intVal":10,"longVal":200}

使用@RequestParam获取参数

Spring Mvc提供注解@RequestParam来确定前后端参数名称的映射关系。

package com.panhao.controller;

import java.util.HashMap;
import java.util.Map;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.ResponseBody;

@Controller
@RequestMapping(value = "/my")
public class MyController {    
    @GetMapping("/annotation")
    @ResponseBody
    public Map<String, Object> requestParam(@RequestParam("int_val")Integer intVal,
                                            @RequestParam("long_val")Long longVal,
                                            @RequestParam("str_val")String str){
        Map<String, Object> paramsMap =new HashMap<String, Object>();
        paramsMap.put("intVal", intVal);
        paramsMap.put("longVal", longVal);
        paramsMap.put("str", str);
        return paramsMap;
    }
}

启动Spring Boot项目之后再浏览器地址栏输入：http://localhost:8080/my/annotation?int_val=1&long_val=2&str_val=str

可以看到页面返回的内容为：{"str":"str","intVal":1,"longVal":2}

注意事项

  1. 此时将请求地址中任意一个参数去掉就会得到异常报错的信息。因为在默认的情况下@RequestParam标注的参数是不能为空的，为了允许它为空，可以在配置它的required属性值为false。

    @RequestParam(value="str_val",required=false)String strVal
传递数组

Spring Mvc内部已经能够支持用逗号分隔的数组参数。

package com.panhao.controller;

import java.util.HashMap;
import java.util.Map;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.ResponseBody;

@Controller
@RequestMapping(value = "/my")
public class MyController {

    @GetMapping("/requestArray")
    @ResponseBody
    public Map<String, Object> requestArray(
            int[] intArr,long[] longArr,String[] strArr){
        Map<String, Object> paramsMap=new HashMap<String, Object>();
        paramsMap.put("intArr",intArr);
        paramsMap.put("longArr", longArr);
        paramsMap.put("strArr",strArr);
        return paramsMap;
    }
}

启动Spring Boot项目在浏览器地址栏输入：http://localhost:8080/my/requestArray?intArr=1,2,3&longArr=4,5,6&strArr=str1,str2,str3

然后在页面中就可以看到：{"intArr":[1,2,3],"strArr":["str1","str2","str3"],"longArr":[4,5,6]}

传递JSON

由于传递JSON的时候需要用到POST的请求所以在开始需要建立一个form表单来作为POST请求的发起者。由于JSP技术是在是老的掉牙，确实不想写JSP页面，所以在此特地配置一下Spring Mvc的静态文件资源路径以及视图解析器。

Spring Boot的Mvc配置主要依赖于类WebMvcConfigurer接口，该接口中有很多的方法都是用来设置MVC的各种配置为。

package com.panhao.configuration;

import org.springframework.context.annotation.Configuration;
import org.springframework.core.Ordered;
import org.springframework.web.servlet.config.annotation.EnableWebMvc;
import org.springframework.web.servlet.config.annotation.ResourceHandlerRegistry;
import org.springframework.web.servlet.config.annotation.ViewControllerRegistry;
import org.springframework.web.servlet.config.annotation.ViewResolverRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;
import org.springframework.web.servlet.view.InternalResourceViewResolver;

@Configuration
//@EnableWebMvc
public class WebConfiguration implements WebMvcConfigurer {

//    配置静态资源访问路径
    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/**").addResourceLocations("classpath:/static/");
        registry.addResourceHandler("/**.html").addResourceLocations("classpath:/static/pages/");
        registry.addResourceHandler("/**.js").addResourceLocations("classpath:/static/");
        registry.addResourceHandler("/**.css").addResourceLocations("classpath:/static/");
    }

//    配置视图解析器
    @Override
    public void configureViewResolvers(ViewResolverRegistry registry) {
        registry.viewResolver(new InternalResourceViewResolver("/pages/", ".html"));
    }
//配置欢迎页面
    @Override
    public void addViewControllers(ViewControllerRegistry registry) {
        registry.addViewController("/").setViewName("index");
        registry.setOrder(Ordered.HIGHEST_PRECEDENCE);
    }
}
  1. addResourceHandlers 是用来配置静态资源访问路径的，一旦Mapping中出现对应的请求，就将其映射到指定的文件夹中。
  2. configureViewResolvers 是用来配置视图解析器的。例如在Controller中返回的为index，实际上返回就是/pages/index.html

通过URL传递参数

在一些网站中，提出了REST风格的，这是参数往往通过URL进行传递。可以通过处理器映射和注解@PathVariable的组合获取URL参数。首先通过处理器映射可以定位参数的位置和名称，而@PathVariable 则可以通过名称来获取参数。

package com.panhao.controller;

@Controller
@RequestMapping("/user")
public class UserController {

    @GetMapping("/url/{id}")
    @ResponseBody
    public Map<String, Object> get(@PathVariable("id")Long id) {
        Map<String, Object>paramsMap=new HashMap<String, Object>();
        paramsMap.put("id", id);
        paramsMap.put("success", true);
        return paramsMap;
    }
}
  1. {id}表示占位符，此时在浏览器输入http://localhost:8080/user/url/1 ，表示的就是id的值为1。
  2. @PathVariable 表示将id的值与方法的参数Long id进行映射。

然后页面点击上个页面中的超链接，就可以在浏览器看到如下的返回信息：{"success":true,"id":200}

@Controller
@RequestMapping("/user")
public class UserController {

    @RequestMapping("/format")
    @ResponseBody
    public Map<String, Object> format(
            @DateTimeFormat(iso = ISO.DATE)Date date,
            @NumberFormat(pattern = "#,###.##")Double moeny){
        Map<String, Object> dataMap=new HashMap<String, Object>();
        dataMap.put("date",date);
        dataMap.put("number", moeny);
        return dataMap;
    }
}

  1. 可以看到前端传输进去的dateTime类型和text类型最后在后端转换之后传过来的就是date格式和double格式。
自定义参数转换规则

Http的请求头包含请求头（Header）、请求体（Body）、URL和参数等内容，服务器还包含其上下文环境和客户端交互会话（Session）机制，而这里的消息转换是请求体的转换。

  1. 处理器获取参数的逻辑

    当一个请求到来的时，在处理器执行的过程中，它首先会从HTTP请求和上下文环境来得到参数。如果是简易的参数它会以简单的转换器进行转换，而这些简单的转换器是Spring Mvc自身已经提供了的。但是如果是转换HTTP请求体（Body），它就会调用HttpMessageConverter接口的方法对请求体的信息进行转换，首先它会判断能否对请求体进行转换，如果可以就将其转换为为Java类型。

    package org.springframework.http.converter;

public interface HttpMessageConverter<T> {
    //是否可读，其中clazz为Java类型，mediaType为Http请求类型
    boolean canRead(Class<?> clazz, @Nullable MediaType mediaType);
    //判断clazz类型是否能够转换为mediaType媒体类型
          //其中clazz是Java类型，mediaType是媒体类型
    boolean canWrite(Class<?> clazz, @Nullable MediaType mediaType);

    //可支持的媒体类型的列表
    List<MediaType> getSupportedMediaTypes();
    //当canRead验证通过后，读入HTTP请求信息
    T read(Class<? extends T> clazz, HttpInputMessage inputMessage)
            throws IOException, HttpMessageNotReadableException;
    //当canWrite方法验证通过以后，写入响应
    void write(T t, @Nullable MediaType contentType, HttpOutputMessage outputMessage)
            throws IOException, HttpMessageNotWritableException;

}
    • 代码中控制器的方法的参数标注了@RequestBody ，所以处理器会采用请求体（Body）的内容进行参数转换，而前端的请求体为JSON类型，所以它首先会调用canRead方法来确定请求体是否可读。如果判定为可读之后，接着就是使用read方法，将前端提交的用户JSON类型的请求体转换为控制器的用户（User）类参数，这样控制器就能够得到参数了。

    在Spring Mvc中，是通过WebDataBinder机制来获取参数的，它的主要作用就是解析HTTP请求的上下文，然后在控制器的调用之前转换参数并其提供验证功能，为调用控制器方法做准备。处理器会从HTTP请求中读取数据，然后通过三种接口来进行各类参数的转换，这三种接口是Converter、Formatter和GenericConverter。

    1. Converter 普通转换器，例如，有一个Integer类型的控制器参数，而从HTTP对应的字符串，对应的Converter就会将字符串转换为Integer类型；
    2. Formatter 格式转换器，类似那些日期字符串就是通过它按照约定的格式转换为日期；
    3. GenericConverter 将HTTP参数转换为数组；
    4. 对于数据类型的转换，Spring Mvc提供了一个服务机制去管理，它就是ConversionService接口。在默认的情况下，会使用这个接口的子类，DefaultFormattingConversionService对象来管理这些类。

  2. 一对一转换器（Converter）

Converter是一对一转换器，也就是从一种类型转换成另一种类型，其接口定义十分简单：

package org.springframework.core.convert.converter;
@FunctionalInterface
public interface Converter<S, T> {
    //转换方法，S代表原类型，T代表目标类型
         @Nullable
    T convert(S source);
}
  • 这个接口的类型有源类型（S）和目标类型（T）两种，它们通过convert方法进行转换。

自定参数规则只需要实现这个接口就可以：

package com.panhao.converter;
@Component
public class StringToUserConverter implements Converter<String, User> {

    @Override
    public User convert(String source) {
        User user=new User();
        String[] strArr=source.split("-");
        String username=strArr[0];
        Integer age=Integer.parseInt(strArr[1]);
        String note=strArr[2];
        user.setUsername(username);
        user.setAge(age);
        user.setNote(note);
        return user;
    }
}
  • 用@Component注解注释，会在项目启动的时候被Spring Boot扫描到Spring Ioc容器中。
数据验证

一般情况下，在参数转换出来之后，紧跟着往往需要验证参数的合法性，因此Spring Mvc也提供了验证机制。一方面，它可以支持JSR-303注解验证，在默认的情况下Spring Boot会引入关于Hibernate Validator机制来支持JSR-303验证规范；另外一方面，因为业务会比较复杂，所有需要自定义验证规则。

  1. JSR-303验证

    JSR-303验证主要是通过注解的方式进行的。这里先定义一个需要验证的POJO，此时需要在其属性中加入相关的注解。当引入spring-boot-web-starter 时，该starter会默认引入Hibernate-validator，也就是Hibernate Validator框架。在pom.xml文件中引入以下依赖：

    <dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-validation</artifactId>
  <version>2.2.2.RELEASE</version>
  <scope>compile</scope>
  <exclusions>
    <exclusion>
      <artifactId>tomcat-embed-el</artifactId>
      <groupId>org.apache.tomcat.embed</groupId>
    </exclusion>
  </exclusions>
</dependency>

    这里需要一个需要被验证的Bean：

    package com.panhao.pojo;

    @NotNull(message = "id不能为空")

   @Future(message = "需要一个将来的日期") //    只能是将来的日期

// @Past 只能是过去的日期
@DateTimeFormat(pattern = “yyyy-MM-dd”) // 日期格式转换
@NotNull // 不能为空
private Date date;

   @NotNull
   @DecimalMin(value = "0.1") //最小值为0.1元
   @DecimalMax(value = "10000.00")//最大值为10000元
   private Double doubleValue =null;

   @Min(value = 1,message = "最小值为1") //最小值为1
   @Max(value = 88,message = "最大值为88") //最小值为88
   @NotNull //不能为空
   private Integer integer;


   @Range(min = 1,max = 88,message = "范围为1-88")//限定范围
   private Long range;

   @Email(message = "邮箱格式错误")
   private String email;

   @Size(min = 20,max = 30,message = "字符串长度要求20-30之间。")
   private String size;

}

   ```java
   package com.panhao.pojo;

       @ResponseBody

               @Valid @RequestBody ValidatorPojo vp,Errors errors){
           Map<String, Object> errMap=new HashMap<String, Object>();
   //        获取错误列表
           List<ObjectError> oes=errors.getAllErrors();
           for(ObjectError oe:oes) {
               String key=null;
               String message=null;
   //            字段错误
               if(oe instanceof FieldError) {
                   FieldError fe=(FieldError) oe;
                   key=fe.getField();//获取错误验证字段名
               }else {
   //                非字段错误
                   key=oe.getObjectName();
               }
   //            错误信息
               String msg=oe.getDefaultMessage();
               errMap.put(key, msg);
           }
           return errMap;

       }
   }

  1. 参数验证机制

    Spring还提供了自己的参数验证机制。在参数转换的时候，可以看到Spring Mvc中，存在WebDataBinder机制进行管理，在默认的情况下Spring会自动地根据上下文通过注册了的转换器转换出控制器所需的参数。在WebDataBinder中除了可以注册转换器外，还允许注册验证器（Validator）。

    package org.springframework.validation;
public interface Validator {

    /**
    判定当前验证器是否支持该Class类型的验证
    @Param clazz --POJO类型
    @return 当前验证器是否支持该POJO类型验证
    **/
    boolean supports(Class<?> clazz);

    /**
    如果support返回true，则这个方法执行验证逻辑
    @param target 被验证的POJO对象
    @param errors 错误对象
    **/
    void validate(Object target, Errors errors);

}

    创建自定义的验证器也是非常简单的一件事情，只需要实现上述接口即可：

    package com.panhao.validator;

import org.springframework.util.StringUtils;
import org.springframework.validation.Errors;
import org.springframework.validation.Validator;

import com.panhao.pojo.User;

public class UserValidator implements Validator{
//    该验证器只支持User类验证
    @Override
    public boolean supports(Class<?> clazz) {
        // TODO Auto-generated method stub
        return clazz.equals(User.class);
    }

    @Override
    public void validate(Object target, Errors errors) {
        // TODO Auto-generated method stub
        if(target==null) {
            errors.rejectValue("", null,"用户不能为空");
            return ;
        }
//        强制转换
        User user=(User) target;
//        用户名非空串
        if(StringUtils.isEmpty(user.getUsername())) {
//            增加错误
            errors.rejectValue("username", null, "用户名不能为空");
        }
    }

}

    此时验证器并不会自动启动它，因为还没有==绑定WebBinder机制==。在Spring Mvc中提供了一个注解@InitBinder 标注的方法。它的作用是在执行控制器之前，处理器会先执行被@InitBinder标注的方法。这时可以将WebDataBinder对象作为参数传递到方法中，通过这层关系得到WebDataBinder对象。这个对象有一个setValidator方法，它可以绑定自定义的验证器。

    package com.panhao.controller;
@Controller
@RequestMapping("/user")
public class UserController {
    @InitBinder
    public void InitBinder(WebDataBinder binder) {
//        绑定验证器
        binder.setValidator(new UserValidator());
//        定义日期参数格式，参数不再需要注解@DataTimeFormat，boolean参数表示允许为空
        binder.registerCustomEditor(Date.class, new CustomDateEditor(new SimpleDateFormat("yyyy-MM-dd"),false));

    }


   @RequestMapping("/valida")
   @ResponseBody
   public Map<String, Object> validator(
           @Valid User user,Errors errors,Date date){
       Map<String, Object> map=new HashMap<String, Object>();
       map.put("user",user);
       map.put("date",date);

// 判断是否存在错误
if(errors.hasErrors()) {
// 获取全部的错误
List oes=errors.getAllErrors();
for(ObjectError oe:oes) {
// 判断是否字段错误
if(oe instanceof FieldError) {
// 字段错误
FieldError fe= (FieldError) oe;
map.put(fe.getField(), fe.getDefaultMessage());
}else {
// 对象错误
map.put(oe.getObjectName(), oe.getDefaultMessage());
}
}
}
return map;
}

}


   * `initBinder` 方法因为标注注解<b style="color:crimson;">@InitBinder</b>，因此会在控制器方法前被执行，并且将`WebDataBinder`对象传递进去，在这个方法里绑定了自定义的验证器`UserValidator`，而且设置了日期的格式，所以在控制器方法中已经不再需要使用<b style="color:crimson;">@DateTimeFormat</b>去定义日期格式化。
