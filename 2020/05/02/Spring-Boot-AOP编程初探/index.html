规划时间是自律的基本能力

Spring Aop

Spring Aop是约定编程中的一种，只要按照一定的规则，就可以将代码织入实现约定的流程中。在Spring中有很多种方式配置AOP，因为Spring Boot中采用注解的方式配置，引入注解@AspectJ。

1、AOP的术语和流程

  • 连接点（join point） ：对应的是具体被拦截的对象，因为Spring只能支持方法，所以被拦截的对象往往就是指特定的方法，AOP将通过动态代理将其织入对应的流程中。
  • 切点（pointcut） ：切面的应用并不局限于单个方法，也可以应用在多个方法上，此时可以通过可以通过正则表达式和指示器的规则去定义，然后来适配连接点。切点就是提供一个这样的功能的概念。
  • 通知（advice）：即是按照约定的流程下的方法，分为：前置通知（before advice）、后置通知（After advice）、环绕通知（around advice）、事后返回通知（afterRunning advice）和异常通知（afterThrowing advice）。它会根据约定织入流程中。
  • 目标对象（target）：即被代理的对象，指的是要被织入切面的类。
  • 引入（introduction）：是指引入新的类和其方法，增强现有Bean的功能。
  • 织入（weaving）：它是一个通过动态代理技术，为原有服务对象生成代理的对象，然后将与切点定义匹配的连接点进行拦截，并按照约定将各类通知织入约定流程的过程
  • 切面（aspect）：是一个可以定义切点、各类通知和引入的内容，Spring Aop将通过它的信息来增强Bean的功能或者将对应的方法织入流程。

2、AOP开发流程详解

步骤一：确定连接点

任何AOP的编程，都实现需要知道在什么地方需要AOP，说白了就是确定连接点（在Spring中，就是什么类什么方法）的问题。

首先定义一个UserService的接口用来作为目标对象：

package com.panhao.demo.aspect.service;

import com.panhao.demo.pojo.User;

public interface UserService {

    public void printUser(User user);
}

然后创建一个UserServiceImpl来实现这个接口：

package com.panhao.demo.aspect.service.impl;

import com.panhao.demo.aspect.service.UserService;
import com.panhao.demo.pojo.User;

public class UserServiceImpl implements UserService{

    public void printUser(User user) {
        // TODO Auto-generated method stub
        if(user==null) {
            throw new RuntimeException("检查用户参数是否为空");
        }
        System.out.println("id="+user.getId());
        System.out.println("username="+user.getUsername());
        System.out.println("note="+user.getNote());
    }

}
步骤二：开发切面

确定了目标对象之后，需要一个切面，通过它可以描述AOP其他的信息，用以描述流程的织入。

package com.panhao.demo.aspect;

import org.aspectj.lang.annotation.After;
import org.aspectj.lang.annotation.AfterReturning;
import org.aspectj.lang.annotation.AfterThrowing;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;

@Aspect
public class MyAspect {

    @Before("execution(* com.panhao.demo.aspect.service.impl.UserServiceImpl.printUser(..))")
    public void before() {
        System.out.println("Befor...");
    }

    @After("execution(* com.panhao.demo.aspect.service.impl.UserServiceImpl.printUser(..))")
    public void after() {
        System.out.println("After...");
    }

    @AfterReturning("execution(* com.panhao.demo.aspect.service.impl.UserServiceImpl.printUser(..))")
    public void afterReturning() {
        System.out.println("afterReturning...");
    }

    @AfterThrowing("execution(* com.panhao.demo.aspect.service.impl.UserServiceImpl.printUser(..))")
    public void afterThrowing() {
        System.out.println("afterThrowing...");
    }

}
  • @Aspect ：Spring是以@Aspect作为切面的声明，当以@Aspect作为注解的时候，Spring就会知道这是一个切面，然后就可以通过各类的通知了。
  • @Before、@After、@AfterReturning、@AfterThrowing ：分别对应着，前置通知，后置通知，事后返回通知，异常通知。
  • execution(* com.panhao.demo.aspect.service.impl.UserServiceImpl.printUser(..)) ：是切点的匹配表达式，要直接匹配方法的层面。
    1. execution 表示在执行的时候，拦截里面的正则匹配的方法；
    2. * 表示任意返回类型的方法；
    3. com.panhao.demo.aspect.service.impl.UserServiceImpl指定目标对象的全限定名称；
    4. pointUser指定目标对象的方法；
    5. (..)表示任意参数进行匹配；

定义切点

如上所述的代码，在@Before、@After、@AfterReturning、@AfterThrowing后面每一都需要书写很长的切点表达式，可以直接定义切点，修改上述代码：

package com.panhao.demo.aspect;

import org.aspectj.lang.annotation.After;
import org.aspectj.lang.annotation.AfterReturning;
import org.aspectj.lang.annotation.AfterThrowing;
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
import org.aspectj.lang.annotation.Pointcut;

@Aspect
public class MyAspect {

    @Pointcut("execution(* com.panhao.demo.aspect.service.impl.UserServiceImpl.printUser(..))")
    public void Pointcut() {

    }

    @Before("Pointcut()")
    public void before() {
        System.out.println("Befor...");
    }

    @After("Pointcut()")
    public void after() {
        System.out.println("After...");
    }

    @AfterReturning("Pointcut()")
    public void afterReturning() {
        System.out.println("afterReturning...");
    }

    @AfterThrowing("Pointcut()")
    public void afterThrowing() {
        System.out.println("afterThrowing...");
    }

}
  • @Pointcut 用来定义切点，它标注在方法pointCut上，在后面的通知注解中就可以使用方法名称定义了。
步骤三：测试AOP

首先搭建一个Web开发环境，开发一个用户控制器UserController：

package com.panhao.demo.aspect.controller;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.ResponseBody;

import com.panhao.demo.aspect.service.UserService;
import com.panhao.demo.pojo.User;

@Controller
@RequestMapping("/user")
public class UserController {

    @Autowired
    private UserService userService=null;

    @RequestMapping("/print")
    @ResponseBody
    public User printUser(Long id,String username,String note) {
        User user=new User();
        user.setId(id);
        user.setUsername(username);
        user.setNote(note);
        userService.printUser(user);
        return user;
    }
}
  • @Controller 表示告诉Spring 这是一个控制器；
  • @ResponseBody ：表示告诉Spring Mvc返回的值转换为JSON格式。

最后修改一下Spring Boot配置文件：

package com.panhao.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.PropertySource;

import com.panhao.demo.aspect.MyAspect;

@SpringBootApplication(scanBasePackages = {"com.panhao.demo.aspect"})
public class MyfirstSpringBootApplication {

    @Bean(name = "myAspect")
    public MyAspect initMyAspect() {
        return new MyAspect();
    }

    public static void main(String[] args) {
        SpringApplication.run(MyfirstSpringBootApplication.class, args);
    }

}

测试的时候在浏览器出输入：[测试AOP]http://localhost:8080/user/print?id=1&username=user_name_1&note=2233

最后在控制台看到输出：

Befor...
id=1
username=user_name_1
note=2333
After...
afterReturning...

如果此时修改UserController对象设置为空：

Befor...
After...
afterThrowing...

可以看到无论是否是发生异常，后置通知都会被流程执行；而因为发生了异常，所以按照约定，异常通知会被触发，返回通知则不会触发。

3、环绕通知

使用环绕通知的场景一般是在需要大幅度修改原有目标服务逻辑的时候，否则都大量使用其他的通知。环绕通知是一个取代原有目标对象方法的通知，当然它也提供回调原有的目标的方法。

    @Around("Pointcut()")
    public void aroud(ProceedingJoinPoint point) throws Throwable {
        System.out.println("aroud brfore...");
        point.proceed();
        System.out.println("aroud after...");
    }

此时再次测试该切面的时候，控制台的输出打印为：

aroud brfore...
Befor...
id=1
username=user_name_1
note=2333
aroud after...
After...
afterReturning...
  • ProceedingJoinPoint类型的对象具有一个方法proceed该方法可以调用原来的逻辑方法。
  • 值得注意的是，在最后打印出来的结果，around brfore在Before的前面。正常我们应该希望Before在aroud brfore之前。这里因为版本的原因。在XML配置的Spring AOP的结果的时候，是按照我们希望的结果，但是Spring Boot使用的时候有一些出入。需要注意合理的使用

4、引入

引入的作用在于当目标对象的业务逻辑需要增加的时候，使用引入增加目标对象的业务逻辑。例如，上述的例子中，增加一个Validator然后测试User内容不为空：

package com.panhao.demo.aspect.validator;

import com.panhao.demo.pojo.User;

public interface UserValidator {

    public boolean validate(User user);
}
package com.panhao.demo.aspect.validator.impl;

import com.panhao.demo.aspect.validator.UserValidator;
import com.panhao.demo.pojo.User;

public class UserValidatorImpl implements UserValidator{

    public boolean validate(User user) {
        // TODO Auto-generated method stub
        System.out.println(    "引入的接口："+UserValidator.class.getSimpleName());
        return user!=null;
    }

}

在MyAspect的切面中增加：

@DeclareParents(value = "com.panhao.demo.aspect.service.impl.UserServiceImpl",defaultImpl = UserValidatorImpl.class)
public UserValidator userValidator;
  • @DeclareParents 的作用是引入新的类来增强服务，它有必须的两个参数value和defaultImpl：
    1. value 指向你要增强功能的目标对象。
    2. defaultImpl 引入增强功能的类。

然后更换一下测试类：

    @RequestMapping("/cp")
    @ResponseBody
    public User validatorAndPrint(Long id,String username,String note) {
        User user=new User();
        user.setId(id);
        user.setUsername(username);
        user.setNote(note);

        UserValidator userValidator=(UserValidator) userService;
        if(userValidator.validate(user)) {
            userService.printUser(user);
        }
        return user;
    }

所以使用UserValidator的validate的方法测试User，然后在执行printUser方法。

在地址栏输入http://localhost:8080/user/cp?id=1&username=user_name_1&note=2232

在控制台出现如下输入：

引入的接口：UserValidator
aroud brfore...
Befor...
id=1
username=user_name_1
note=2333
aroud after...
After...
afterReturning...

可以很显然的看出来，UserValidator的validate方法在printService之前执行。实现增强功能。

5、通知获取参数

通知依旧是可以获取参数的，对于非环绕通知还可以使用一个连接点（JointPoint）类型的参数，通过它也可以获取参数。 

@Before("Pointcut() && (user)")
public void beforeParam(JoinPoint point,User user){
    Object [] args=point.getArgs();
    System.out.println("before...")
}
  • pointcut()&&args(user)，pointCut()表示启用原来定义切点的规则，并且约定将连接点（目标对象方法）名称为user的参数传递进来。JointPoint类型的参数对于非环绕的通知而言，Spring AOP会自动的把它传递到通知中；对于环绕通知而言，可以使用ProceedingJoinPoint类型的参数。

6、织入

织入是一个生成动态代理的对象并且将切面和目标对象方法编织成约定流程的过程。对于动态代理也有很多的实现方式，JDK、CGLIB、Javassist、ASM等。Spring采用了JDK和CGLIB，对于JDK而言，它是要求被代理的目标对象必须拥有接口，而对于CGLIB则不作要求。因此默认的情况下，Spring会按照这样的一条规则进行处理，即当你需要使用AOP的类拥有接口时，它会以JDK动态代理运行，否则使用CGLIB代理执行。

将UserServiceImpl修改为不实现接口的类：

@Service
public class UserServiceImpl{

    public void printUser(User user) {
        // TODO Auto-generated method stub
        if(user==null) {
            throw new RuntimeException("检查用户参数是否为空");
        }
        System.out.println("id="+user.getId());
        System.out.println("username="+user.getUsername());
        System.out.println("note="+user.getNote());
    }

}

然后修改一下控制器依赖注入，直接依赖于不存在接口的实现类：

@Controller
@RequestMapping("/user")
public class UserController {

    @Autowired
    private UserServiceImpl userService=null;

    @RequestMapping("/print")
    @ResponseBody
    public User printUser(Long id,String username,String note) {
        User user=new User();
        user.setId(id);
        user.setUsername(username);
        user.setNote(note);
        userService.printUser(user);

        return user;
    }
}

控制台输出为：

aroud brfore...
Befor...
id=1
username=user_name_1
note=2333
aroud after...
After...
afterReturning...

7、多个切面

不光是单个切面的编程，Spring依旧支持多个切面的运行。在组织多个切面的时候需要知道其运行的顺序。

定义切面的顺序的可以放在Spring Boot的配置文件中：

package com.panhao.demo;
@SpringBootApplication(scanBasePackages = {"com.panhao.demo.aspect"})
public class MyfirstSpringBootApplication {

    @Bean(name = "myAspect2")
    public MyAspect initMyAspect() {
        return new MyAspect();
    }

      @Bean(name = "myAspect1")
    public MyAspect initMyAspect() {
        return new MyAspect();
    }

    @Bean(name = "myAspect3")
    public MyAspect initMyAspect() {
        return new MyAspect();
    }

    public static void main(String[] args) {
        SpringApplication.run(MyfirstSpringBootApplication.class, args);
    }

}

然后就可以在控制台看到如下输出：

MyAspect2 before...
MyAspect1 before...
MyAspect3 before...
 测试多个切面的顺序
MyAspect3 after...
MyAspect3 afterReturning...
MyAspect1 after...
MyAspect1 afterReturning...
MyAspect2 after...
MyAspect2 afterReturning...

同样的，可以使用Ordered接口也可以指定顺序。推荐使用@Order注解。

@Aspect
public class MyAspect1 implements Ordered{

    public int getOrder(){
        return 1;
    }
}

默认的是值越小优先级越高。

@Aspect
@Order(1)
public class MyAspect1 {

}
